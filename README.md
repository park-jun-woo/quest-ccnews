# ccnews

뉴스 기사에서 **육하원칙(event6)** 을 추출하는 **퀘스트 CLI**.

> **퀘스트란?** 완료 조건이 명확하고 기계가 검증하는 작업 단위.
> AI가 "요약했어요"라고 말한다고 끝나지 않는다 — 추출한 사실이 **원문에
> 실제로 있는지** 기계가 대조해 통과한 것만 잠근다. 완료 판정 권한은 AI에게 없다.

> 생성은 AI가, 판정은 기계가.

## 왜 만드나 (공개 가능한 퀘스트 데모)

회사 이메일 수집([comail](https://github.com/park-jun-woo/quest-comail))은 무단수집·개인정보
이슈가 있었다. ccnews는 **공개 가능한** 퀘스트 데모다:

- **수집 적법성** — robots.txt가 허용한 사이트만 크롤(거부 사이트는 스킵·감사기록).
- **공개 적법성** — 결과는 *사실(육하원칙)* 만 농축. **사실은 저작권 비보호** → 원문 링크와 함께 공개 안전.

## 구조 (2층 퀘스트 + 캐시)

```
호스트 단위 (캐싱):
  1) robots 게이트       robots.txt 파싱 → 우리 UA·경로 Allow? (호스트당 1회)
  2) 추출 템플릿 게이트   JSON-LD/OG(공짜) → 사이트별 셀렉터 템플릿(캐싱) → AI(미스 시 생성·재실행 검증)

기사 단위:
  본문 추출 → AI가 event6 추출 → 사실 앵커 게이트 → PASS/REVIEW
```

### 추출 캐스케이드 (토큰 절약)
| 단계 | 비용 | 방식 |
| --- | --- | --- |
| 1 | 0토큰 | JSON-LD `schema.org/NewsArticle` + OpenGraph 메타 |
| 2 | 캐시 | 등록된 CSS 셀렉터 템플릿(드리프트 시 갱신) |
| 3 | AI | 1·2 미스 → 에이전트가 템플릿 생성 → **재실행 검증 후 등록** |

### 사실 앵커 게이트 (환각 차단)
- **필수: 누가·언제·무엇** — 값 + 원문 토큰 앵커 필수(없으면 FAIL).
- **선택: 어디서·어떻게·왜** — 있으면 앵커 검증, 없으면 빈칸 허용.
- 해석이 섞인 필드는 AI가 자기 표현으로 짧게 paraphrase(핵심 사실 토큰은 앵커). 문장 통째 복제 금지(표현 저작권 회피).

## 수집 항목 (스키마, 잠정)

| 그룹 | 컬럼 |
| --- | --- |
| 출처/증빙 | 매체명 · 매체사이트 · 원문URL · 발행일 · 수집일시 |
| event6 | 누가 · 언제 · 어디서 · 무엇 · 어떻게 · 왜 (존재하는 것만) |
| 적법성 라벨 | 크롤링허용 · CC라이선스종류 |
| 퀘스트 판정 | 검증상태(PASS/REVIEW) |
